MVP셔틀버스 관리(GPS 운행이력)

1차 오픈(Minimum Viable Product) 스펙 전용 문서

🚀 MVP 스펙 아웃(Spec-Out) 항목 안내

초기 런칭 기준 관리자가 '최소한'으로 차량 동선을 확인하고 로그를 확인하기 위해, 다음과 같은 개발 및 리소스 투입이 큰 기능은 오픈 스펙에서 제외(혹은 단순화)시킵니다.

  • 리버스 지오코딩 (위치 주소 매칭) ➔ 지도 시각화로 대체하여 불필요한 API 호출 비용 제거
  • 체류시간, 장기정차, 과속 시각화 로직 ➔ 추후 고도화 시 적용, MVP에선 Raw 속도(km/h) 수치만 그대로 노출
  • 점선 보간, 가상 스크롤, 맵 툴팁 ➔ 일단은 테이블에 띄울 로그 수량을 조절(예: 5분 간격 샘플링)하거나 기본 렌더링으로 오픈

MVP 핵심 화면 개요

관리자가 지정한 날짜/시간의 차량 GPS 센서 데이터를 조회하여, 지도 상에 이동 경로 라인을 그리고 하단에는 해당하는 로그 데이터를 리스팅하는 가장 기본적인 관제 뷰입니다.

화면 구성 요소

  • 1

    조회 필터 바

    단일 운행날짜, 시간 범위, 특정 차량을 선택하고 [조회] 버튼을 클릭. 조회된 내역은 CSV(엑셀) 파일로 다운로드 가능합니다.

  • 2

    심플 지도 경로 표기

    카카오맵 API의 Polyline(선 긋기) 도구를 사용하여 수신된 위경도 로그 포인트들을 단순히 직선으로 이어 이동 궤적을 확인합니다.

  • 3

    마커 연동

    모든 로그에 마커를 찍지 않고 경로 선만 그리게 하여 브라우저 렌더링 부하를 줄입니다. 사용자가 테이블 행(Row)을 선택했을 당시에만 파란색 마커 1개가 렌더링되어 해당 위치를 지시합니다.

  • 4

    데이터 테이블

    복잡하게 가공하지 않은 [순번, 차량ID, 기록시간, GPS 속도] 등 원천 데이터 위주로 리스팅합니다.

🎯 Next: 고도화(Phase 2) 방향

MVP 이후 도입될 정식 버전의 핵심 고도화 스펙입니다. MVP에서 발생할 수 있는 데이터 신뢰도와 브라우저 과부하 문제를 차츰 해결합니다.

  • 빅데이터 한계 극복: 가상 스크롤, 맵 클러스터링을 통한 대용량 렌더링 성능 확보
  • 상태 분석 및 경고: 과속 알림, 이상 정차, 위치명(Reverse Geocoding) 자동 변환 등
  • 보정 알고리즘: 터널/음영 공간 결측치 보간(점선), 튀는 값(Noise) 자동 제거 전처리
