MVP셔틀버스 관리(GPS 운행이력)

1차 오픈(Minimum Viable Product) 스펙 전용 문서

MVP 스펙 아웃(Spec-Out) 항목 안내

초기 런칭 기준 관리자가 '최소한'으로 차량 동선을 확인하고 로그를 확인하기 위해, 다음과 같은 개발 및 리소스 투입이 큰 기능은 오픈 스펙에서 제외(혹은 단순화)시킵니다.

  • 리버스 지오코딩 (위치 주소 매칭) → 지도 시각화로 대체하여 불필요한 API 호출 비용 제거
  • 체류시간, 장기정차, 과속 시각화 로직 → 추후 고도화 시 적용, MVP에선 Raw 속도(km/h) 수치만 그대로 노출
  • 점선 보간, 가상 스크롤, 맵 툴팁 → 일단은 테이블에 띄울 로그 수량을 조절(예: 5분 간격 샘플링)하거나 기본 렌더링으로 오픈

MVP 핵심 화면 개요

관리자가 지정한 날짜/시간의 차량 GPS 센서 데이터를 조회하여, 좌측 지도에 이동 경로를 표시하고 우측 테이블에 로그를 리스팅하는 관제 뷰입니다.

화면 구성 요소

  • 1

    조회 필터 바

    운행 날짜, 출발 시간, 조회 간격, 차량 선택을 지정하고 [조회] 버튼을 클릭. 조회된 내역은 [Excel 다운로드] 버튼으로 내려받을 수 있습니다.

  • 2

    지도 영역 (좌측)

    카카오맵 API의 Polyline으로 이동 궤적을 표시합니다. 테이블에서 행을 선택하면 해당 위치에 마커와 툴팁이 표시됩니다.

  • 3

    마커/툴팁 연동

    테이블 행(Row) 선택 시 지도에 마커 1개가 렌더링되고, BUS ID / 시간 / 속도를 표시하는 툴팁이 노출됩니다.

  • 4

    데이터 테이블 (우측)

    3개 컬럼(No, 기록 시간, 속도)으로 단순 구성합니다. 상단에 총 건수, 페이지 사이즈 설정, 페이지 네비게이션을 제공합니다.

Next: 고도화(Phase 2) 방향

MVP 이후 도입될 정식 버전의 핵심 고도화 스펙입니다.

  • 빅데이터 한계 극복: 가상 스크롤, 맵 클러스터링을 통한 대용량 렌더링 성능 확보
  • 상태 분석 및 경고: 과속 알림, 이상 정차, 위치명(Reverse Geocoding) 자동 변환 등
  • 보정 알고리즘: 터널/음영 공간 결측치 보간(점선), 튀는 값(Noise) 자동 제거 전처리
