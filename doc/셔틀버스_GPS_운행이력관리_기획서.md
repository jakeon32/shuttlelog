# 셔틀버스 GPS 운행 이력관리 화면 기획 v1.0

## 1. 화면 목적 및 개요

- **목적:** 특정 운행일과 시간 범위를 선택해, 셔틀 차량의 GPS 로그 이력(위치, 시간, 속도)을 조회·분석하고, 필요 시 개별 로그를 지도에서 확인할 수 있도록 한다.
- **대상 사용자:** 셔틀 운영 관리자(관제, 민원 응대, 사고/지연 이슈 확인 등).
- **주요 특징:**
  - 지도는 시각화/맥락용 서브 뷰, 테이블은 선택·탐색용 메인 뷰.
  - 최소 컬럼 구성(로그번호, 차량 ID, 시간, 속도)으로 단순·명료하게 설계.
  - 마커 겹침 문제는 "테이블 중심 인터랙션 + 지도 연동"으로 근본적으로 완화.

## 2. IA 및 메뉴 구조

- 좌측 사이드바 (기존 구조 유지)
  - 대시보드
  - 공지사항 관리
  - 셔틀버스 위치보기 (실시간)
  - **운행이력 관리 (신규)**
  - 통계
  - 긴급공지 관리

- `운행이력 관리` 선택 시, 우측 메인 영역에 **이력관리 화면** 표시.

---

## 3. 레이아웃 구조

기본 레이아웃은 상단 필터, 중앙 지도, 하단 테이블 3단 구성.

```text
┌────────────────────────────────────────────────────────────┐
│ 상단 컨트롤 바 (필터 영역)                                  │
├────────────────────────────────────────────────────────────┤
│ 지도 영역 (GPS 경로, 마커)                                 │
├────────────────────────────────────────────────────────────┤
│ 로그 테이블 영역 (메인 인터랙션)                            │
└────────────────────────────────────────────────────────────┘
```

- 1차안 비율: 지도 40%, 테이블 60% (세로 방향).
- 해상도: PC 관제용(데스크톱 우선), 모바일 대응은 후순위.

---

## 4. 상단 컨트롤 바 설계

### 4.1 필터 요소

1. **운행 날짜(Date Picker)**
    - 단일 날짜 선택.
    - 기본값: 오늘 날짜.
2. **시간 선택 및 조회 간격(Time & Interval)**
    - GPS 로그(5초 주기)의 렌더링 부하를 줄이고 시인성을 높이기 위해 시작 시간과 조회 간격을 선택.
    - **시작 시간:** 30분 단위 선택 (e.g. `07:00`, `07:30`, `08:00`). 기본값은 `시간을 선택해주세요`.
    - **간격 선택:** `30분`, `1시간` 중 선택. 기본값은 `30분`.
3. **차량 선택(Vehicle Select)**
    - 드롭다운에서 차량 ID 검색·선택.
    - 1차 버전: 단일 선택(복수 선택은 추후 확장).
4. **조회 버튼**
    - 클릭 시 지도와 테이블이 선택 조건에 맞게 새로고침.

### 4.2 요약 인포 바(옵션)

조회 후, 바로 아래에 간단한 요약 지표 표시.

- 선택 차량 수
- 선택 기간 내 총 로그 수
- 평균 속도, 최대 속도 등 (추후 필요 시).

## 5. 지도 영역 설계

### 5.1 기본 표시

- Kakao Map (실시간 위치보기 화면과 동일 베이스).
- 선택한 날짜/시간 범위/차량에 해당하는 GPS 포인트를 **폴리라인 경로 + 포인트 마커**로 표시.
- 경로 색상:
    - 단일 차량일 경우: 노선 컬러 한 가지.
    - 다중 차량(차후 확장) 시: 차량별 색상 분리.

### 5.2 마커 및 경로 인터랙션

- 테이블 행 hover/클릭 시, 해당 GPS 포인트 마커를:
    - 확대, 강조(두꺼운 테두리, 상단 z-index).
    - 지도 중심으로 스무스하게 이동.
- 선택된 포인트까지의 경로는 **진하게**, 나머지는 **연하게** 표현해 "지금까지의 이동" 흐름을 보여줌.

### 5.3 겹침 문제에 대한 기본 대응

1. **테이블 우선 선택 구조**
    - 사용자는 주로 테이블에서 로그를 선택하고, 지도는 그 결과를 보여주는 역할을 한다.
    - 마커가 겹쳐도 테이블에서 항상 정확한 로그를 선택할 수 있으므로, 정보 접근이 막히지 않는다.
2. **간단 겹침 리스트 팝업(2차 확장)**
    - 동일 좌표/근접 포인트를 클릭했을 때, 해당 지점의 로그 리스트를 작은 팝업으로 표시 후 하나 선택하는 방식.
3. **클러스터링(중/장기)**
    - 줌 레벨이 낮을 때는 여러 포인트를 하나의 클러스터 원(숫자 배지)로 묶고, 줌인 시 개별 포인트로 분해하는 패턴을 고려.

## 6. 로그 테이블 영역 설계

### 6.1 역할 정의

- 이 화면의 **메인 인터랙션** 영역.
- 원하는 시점/로그를 빠르게 찾고, 선택한 로그에 대해 지도에서 위치·경로를 확인하는 역할.

### 6.2 컬럼 구성 (1차 최소 구성)

| 필드명 | 예시 | 설명 |
| :-- | :-- | :-- |
| 로그 번호 | 123 | 선택 기간 내 시퀀스 번호 |
| 차량 ID | BUS-001 | GPS 데이터를 보낸 차량 ID |
| 시간 | 08:14:32 | 로그 기록 시각 |
| 속도 | 36km/h | 해당 시점 GPS 기반 순간 속도 |

- 기본 정렬: 시간 오름차순.
- 헤더 클릭 시 정렬 토글: 시간 / 속도 기준 정렬 지원.

### 6.3 테이블 인터랙션

- **행 hover:**
    - 지도에서 해당 포인트 마커에 하이라이트 효과.
- **행 클릭:**
    - 지도 중심 이동 + 해당 포인트 툴팁 표시(차량 ID, 시간, 속도, 필요시 위치명 등).
- 페이징:
    - 기본 50건/페이지(추후 조정 가능).
- 검색/필터(옵션):
    - 상단에 간단한 텍스트 검색(차량 ID, 시간 문자열 등) 옵션 고려.

### 6.4 확장 가능한 추가 정보(향후 옵션 컬럼)

- 위치명(역/정류장 이름, 지오코딩 결과).
- 상태(운행, 정차, 장시간 정차 등).
- 체류 시간.
- 방향(Heading) 또는 고도(Altitude) 등 고도화 데이터.

## 7. 지도-테이블 양방향 연동 플로우

1. **조회 → 데이터 로드**
    - 상단 필터에서 날짜·시간 범위·차량 선택 후 [조회].
    - 서버에서 해당 조건의 GPS 로그 리스트를 조회, 지도와 테이블에 동시에 바인딩.
2. **테이블 → 지도**
    - 사용자가 특정 행을 클릭:
        - 해당 GPS 포인트를 중심으로 지도 이동.
        - 마커 강조 + 간단 정보 툴팁 표시.
    - 여러 행을 키보드 + 클릭으로 선택하는 경우, 선택된 구간만 경로로 강조(2차 확장).
3. **지도 → 테이블**
    - 마커 클릭:
        - 테이블에서 해당 로그 행으로 자동 스크롤 후 하이라이트.
    - 마커가 여러 개 겹쳐 있을 경우(2차 기능):
        - "이 위치의 로그 n건" 팝업 > 리스트에서 선택 > 테이블 행으로 포커스 이동.

## 8. 상태 및 피드백 설계

### 8.1 로딩 상태

- [조회] 후 데이터 로딩 중:
    - 지도에 Skeleton/로딩 스피너.
    - 테이블 상단에 "데이터를 불러오는 중입니다…" 메시지 표시.

### 8.2 Empty State

- 지정한 조건에 해당 로그가 없을 때:
    - 테이블에 "해당 조건에 맞는 운행 이력이 없습니다" 안내.
    - 시간 범위, 차량, 날짜를 다시 확인하라는 간단한 가이드 텍스트.

### 8.3 에러 상태

- 서버 에러, 네트워크 오류 시:
    - 토스트 또는 인라인 에러 메시지.
    - 재시도 버튼.

---

## 9. 기술·데이터 고려사항 (간단 메모)

- GPS 로그 데이터 예시 스키마:
    - `id`, `vehicle_id`, `timestamp`, `lat`, `lng`, `speed`, `heading`, `status` 등.
- API 예시:
    - `GET /api/logs?vehicleId=BUS-001&date=2026-03-11&from=07:00&to=10:00`
- 클라이언트:
    - 필터 상태 관리, 응답 데이터 캐싱(시간 범위 조정 시 재활용) 검토.

---

## 10. 향후 확장 아이디어

1. **타임라인/리플레이 뷰**
    - 지도 아래에 슬라이더 기반 타임라인을 두고, 재생 버튼으로 경로 애니메이션.
2. **이상 운행 감지**
    - 과속, 장시간 정차, 경로 이탈 등 이벤트를 별도 아이콘/색상으로 표기.
3. **보고서/내보내기**
    - 현재 테이블 데이터를 CSV/Excel로 다운로드.
4. **멀티 차량 비교**
    - 동일 시간대 여러 차량을 색상 구분하여 경로 비교.
